# 박다훈 개인 프로필 홈페이지 — 사용 안내

이 가이드는 네 가지를 설명합니다.

1. GitHub Pages로 웹에 올리기 (최초 1회)
2. 프로필 사진 추가하기
3. 내용 수정하기
4. 수정 후 웹에 반영 확인

모두 **웹 브라우저만으로** 가능하며, 터미널·Git 명령어·개발 도구를 쓰지 않습니다.

## 파일 구성

```
index.html      프로필 페이지 본체 (이 한 파일에 모든 내용이 들어있음)
profile.jpg     나중에 추가할 사진 (같은 폴더에 두면 자동 적용)
```

---

## 1. 웹에 올리기 — GitHub Pages

### 1-1. GitHub 계정 만들기 (이미 있으면 건너뛰기)

1. <https://github.com> 접속
2. 우측 상단 **Sign up** 클릭
3. 이메일, 비밀번호, 유저명 입력 — **유저명이 그대로 웹 주소에 들어가니 원하는 이름으로 설정** (예: `dahoonpark`)
4. 이메일 인증 완료

### 1-2. 저장소(Repository) 만들기

1. GitHub 우측 상단 **+** 버튼 → **New repository**
2. **Repository name**: `{유저명}.github.io` 형식으로 입력
   - 예: 유저명이 `dahoonpark` 이면 `dahoonpark.github.io`
   - 이 형식으로 만들면 주소가 `https://dahoonpark.github.io/` 로 깔끔하게 나옵니다
3. **Public** 선택 (Private은 유료 플랜에서만 Pages 사용 가능)
4. **Add a README file** 체크 (선택, 있어도 없어도 상관 없음)
5. 아래 **Create repository** 클릭

### 1-3. index.html 업로드

1. 방금 만든 저장소 페이지에서 **Add file** → **Upload files**
2. `index.html` 파일을 브라우저 창에 드래그해서 올리기
3. 아래 **Commit changes** 초록 버튼 클릭

### 1-4. Pages 활성화

1. 저장소 상단 탭에서 **Settings** 클릭
2. 왼쪽 메뉴 아래쪽에 있는 **Pages** 클릭
3. **Source** 섹션에서:
   - Branch: `main` 선택
   - Folder: `/ (root)` 선택
   - **Save** 클릭
4. **1~2분** 기다리면 같은 페이지 상단에 "Your site is live at `https://{유저명}.github.io/`" 문구가 뜹니다
5. 그 주소를 클릭하면 프로필 페이지가 열립니다

이 주소가 영구 주소입니다. 명함, 이메일 서명, 학회 발표 자료 등에 바로 쓸 수 있습니다.

## 2. 프로필 사진 추가하기

1. 3:4 세로 비율 사진을 준비 (이력서 증명사진 또는 프로필용 사진 권장)
2. 파일명을 정확히 **`profile.jpg`** (전부 소문자)로 저장
3. GitHub 저장소 메인 페이지에서 **Add file** → **Upload files**
4. `profile.jpg` 드래그해서 올리기
5. 아래 **Commit changes** 클릭
6. 약 1분 후 웹사이트를 새로고침하면 "박" 이니셜 자리에 사진이 자동으로 표시됩니다

파일명은 정확히 `profile.jpg`여야 합니다. `Profile.jpg`(대문자 P) 또는 `profile.jpeg`는 인식되지 않습니다.

다른 파일명을 쓰고 싶다면 `index.html`에서 아래 줄을 찾아 수정하세요.

```html
<img src="profile.jpg" alt="박다훈" onerror="this.style.display='none'">
```

## 3. 내용 수정하기 — GitHub 웹 편집기

터미널이나 로컬 편집기 없이, 브라우저 안에서 바로 수정할 수 있습니다.

### 3-1. 편집 시작

1. GitHub 저장소에서 `index.html` 파일 클릭
2. 우측 상단 **연필 아이콘(Edit)** 클릭
3. 브라우저 안에서 코드 편집기가 열림
4. `Ctrl + F` (Mac: `Cmd + F`)로 수정할 부분 검색

### 3-2. 자주 수정할 만한 부분

| 수정 내용 | 검색할 키워드 / 찾을 부분 | 설명 |
|---|---|---|
| 이메일 | `doslaza@naver.com` | 검색해서 수정 |
| 소개 문구 | `hero-tagline` | 히어로 영역의 한 문단 |
| 논문 추가 | `pub-list` | 기존 `<li>` 하나를 복사해서 붙여넣고 내용 변경 |
| 학회 발표 추가 | `pres-list` | 기존 `<li>` 복사 |
| 연구용역 추가 | `proj-row` | 기존 `<div class="proj-row">` 복사 |
| 경력 추가 | `tl-list` | Experience / Education 각각 |
| 통계 숫자 (14, 7, 1, 3) | `hero-stats` | 히어로 하단의 4개 숫자 |
| 요약 숫자 (14건, 30.87억원 등) | `summary-strip` | 각 섹션 상단 요약 |
| 업데이트 날짜 | `Last updated April 2026` | 푸터 |

### 3-3. 예: 새 논문 한 편 추가하기

1. 연필 아이콘 클릭해서 편집 모드 진입
2. `Ctrl + F` → `pub-list` 검색해서 Publications 섹션으로 이동
3. 가장 최근 논문 블록 (`<li class="reveal">` 부터 `</li>` 까지) 전체를 복사
4. 그 `</li>` 바로 아래에 붙여넣기
5. 새 블록 안에서 연도·제목·저자·학술지·태그(KCI/SCI(E))를 수정
6. 페이지 하단 **Commit changes** 클릭 (메시지 칸은 "2026년 신규 논문 추가" 처럼 본인이 알아볼 내용으로 입력해도 되고 기본값 그대로 둬도 됩니다)
7. 약 1분 후 사이트 새로고침하면 반영됨

### 3-4. 실수했을 때 되돌리기

GitHub는 모든 변경 이력을 자동 저장하므로 얼마든지 되돌릴 수 있습니다.

1. 저장소 메인 페이지에서 `index.html` 클릭
2. 우측 상단 **History** 버튼
3. 돌아가고 싶은 시점의 커밋 클릭
4. 그 버전의 파일 내용을 복사해서 현재 파일에 덮어쓰기

---

## 4. 수정 후 웹 반영 확인

- 커밋 후 **30초 ~ 2분** 안에 실제 사이트에 반영됩니다
- 반영이 안 보이면 브라우저에서 **강력 새로고침**: `Ctrl + Shift + R` (Mac: `Cmd + Shift + R`)
- 저장소 **Actions** 탭에서 배포 상태를 확인할 수 있습니다 (녹색 체크 = 배포 완료)

## 자주 나오는 질문

**Q. 로컬에서 미리보기 하려면요?**
A. `index.html` 파일을 더블클릭하면 기본 브라우저에서 바로 열립니다. 별도 서버·설치 필요 없음.

**Q. 다른 사람이 내 페이지를 수정할 수 있나요?**
A. Public 저장소라도 직접 수정은 저장소 소유자(본인)만 가능합니다. 다른 사람은 Pull Request 제안만 할 수 있고 본인이 승인하지 않으면 반영되지 않습니다.

**Q. 수정할 때마다 GitHub에 다시 올려야 하나요?**
A. 네, 하지만 전체 재업로드는 필요 없습니다. 웹 편집기(3-1 참고)로 해당 파일만 직접 수정·저장하면 됩니다. 또는 파일만 교체해서 재업로드도 가능.

**Q. 주소를 커스텀 도메인(예: parkdahoon.com)으로 바꾸고 싶어요.**
A. 도메인을 구매한 후 (Gabia, Namecheap 등) 저장소 **Settings → Pages → Custom domain**에 입력하고 DNS 설정하면 됩니다. 당장 필요 없다면 기본 주소로도 충분.

**Q. 모바일에서도 잘 나오나요?**
A. 반응형 디자인이라 모바일·태블릿·데스크톱 자동 대응합니다. 휴대폰으로 주소 열어서 확인해보시면 됩니다.

**Q. 청주대학교 개인 홈페이지 공간을 쓰는 것과 어떤 차이가 있나요?**
A. 둘 다 가능하지만 GitHub Pages가 속도·안정성·편집 편의성 모두 낫습니다. 대학 공간은 소속 변경 시 주소가 사라질 수 있는 반면 GitHub은 영구적입니다.

## 참고: 다른 호스팅 옵션

### Netlify Drop — 1분 즉시 배포

가장 빠르게 테스트해보고 싶을 때. <https://app.netlify.com/drop> 에서 `index.html` 드래그만 하면 즉시 URL 생성. 단, 수정할 때마다 다시 드래그해야 하고 주소도 랜덤으로 나와서 장기 사용엔 GitHub Pages보다 불편합니다.

### Vercel

GitHub 저장소와 연결하면 커밋 시 자동 배포. GitHub Pages보다 반영 속도가 약간 빠릅니다. <https://vercel.com>
